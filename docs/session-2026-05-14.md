   - 原因: LLM 调用期间 session 可能继续增长，若结束时用最新 total 推进指针，会跳过未提取消息

4. 测试脚本固定等待不稳
   - 修复: e2e 改为轮询 `last_consolidated > 0`，最多等待 15s
